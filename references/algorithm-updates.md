# Google Algorithm Updates — SearchLens

## How to Use This File

When a page shows a sudden ranking drop or gain, cross-reference the date of the change with the updates below. A drop within 2 weeks of a confirmed update is likely algorithm-related. A drop with no nearby update points to technical issues, content changes, or competitor gains.

**Correlation threshold:** Flag as "possible algorithm impact" if the drop/gain occurred within 14 days before or after the update's rollout start date.

## Recurring Pattern: What Each Update Type Targets

### Core Updates
Broad reassessment of page quality. No single fix. Patterns in impacted sites:
- Thin or shallow content relative to competitors
- Low E-E-A-T signals (no author, no credentials, no original research)
- Over-reliance on exact-match keywords
- Poor user engagement metrics

**Recovery time:** 3–6 months minimum (need to wait for next core update to see full recovery)

### Helpful Content Updates (pre-2024)
Targeted sites where the primary purpose is search engine traffic, not user value.
- Content that answers no real question
- AI-generated content published without human review
- Sites with predominantly SEO-optimized content vs. genuine subject matter expertise

**Recovery:** Requires substantial content overhaul + some sites have not recovered

### Spam Updates
Target link schemes, scaled content abuse, and technical spam.
- Unnatural inbound link profiles
- PBNs, link farms, paid links
- Mass low-quality content published at scale
- Expired domain abuse (buying expired domains to redirect authority)

**Recovery:** Disavow + link cleanup + content cleanup. Can take 6–12 months.

### Reviews Updates
Target thin review content.
- Reviews that don't demonstrate first-hand use of the product
- Affiliate content without genuine testing
- Generic pros/cons without specifics

**Recovery:** Rewrite reviews with original testing, photos, specific findings

## Diagnosis Framework

When a drop correlates with an update:

```
Step 1: Identify the update type (Core / Helpful Content / Spam / Reviews)
Step 2: Check which patterns that update targets (see above)
Step 3: Audit the impacted pages against those patterns
Step 4: Classify the issue:
  - Content quality → rewrite with more depth, expertise, original data
  - E-E-A-T → add author bios, credentials, first-person experience signals
  - Link profile → audit inbound links, disavow if necessary
  - Technical spam → fix crawl issues, canonicalization, indexation problems
Step 5: Estimate recovery timeline (Core = next Core update; Spam = 3–6 months post-cleanup)
```

## Non-Algorithm Drop Causes (When No Update Correlates)

If the drop doesn't correlate with any known update:

| Symptom | Likely cause |
|---|---|
| Single page drops, others stable | Content change on that page / internal links removed |
| Multiple pages in same cluster drop | Cannibalization introduced / pillar page changed |
| Site-wide drop overnight | Technical issue (crawl block, noindex added, redirect loop) |
| Gradual decline over 60–90 days | Competitors improving / link equity lost / content aging |
| Drop in branded queries | Brand search volume declined (external factor) |
| Seasonal pattern | Normal seasonality — compare to same period last year |
